### Step 1: Test PHP Mail Configuration
Visit: `http://localhost:8000/test_email.php`

This will:

2. Allow you to send a test email

**To send test email:**
- Enter your email address
- Click "Send Test Email"
- Check inbox and spam folder
- If received → Email system is working ✓

### Step 2: Test Full Enrollment Flow

**From Your Browser Console:**

1. Open `http://localhost:8000/enrollment.html`
2. Fill in the form:
   - Name: John Doe
   - Email: your-email@example.com
   - Phone: 555-1234
   - City: Your City
   - Company: Your Company
   - Course: Select any course
   - Click "Proceed to Payment"

3. You should be redirected to PayPal or success page
4. On success page, open **Browser DevTools** (F12)
5. Go to **Console** tab
6. Paste this code:
   ```javascript
   const data = JSON.parse(localStorage.getItem('adeptskil_enrollment_data'));
   console.log('Data in localStorage:', data);
   ```
7. Check what data was stored (should see all fields)

### Step 3: Verify Email Was Sent

**Check Email Logs:**
- Navigate to: `emails/` folder in project root
- Look for files:
  - `confirmation_log.txt` - Log of all sending attempts
  - `YYYYMMDDHHMMSS_StudentName.html` - Backup confirmation email

**Check confirmation_log.txt for:**

## 🔍 Debugging Guide

### Issue 1: Course Name Not Showing on Success Page

**Root Cause:** Data not in localStorage or field name mismatch

**To Debug:**
1. Open success.html in browser
2. Press F12 to open DevTools
3. Go to Application → LocalStorage
4. Find key: `adeptskil_enrollment_data`
5. If not found → enrollment form isn't saving data
6. If found → Check if `course` field has a value

**Fix:**
- Ensure enrollment.html is saving: `localStorage.setItem('adeptskil_enrollment_data', ...)`
- Verify course field is selected before submission
- Check browser console for JavaScript errors

### Issue 2: Date Not Showing on Success Page

**This should always work** as it's generated by: `new Date().toLocaleString()`

**If not showing:**
1. Check browser console (F12) for errors
2. Verify JavaScript is not disabled
3. Check if page is loading properly

### Issue 3: Emails Not Sent

**Possible Causes:**

1. **PHP mail() not configured**
   - Test via: `http://localhost:8000/test_email.php`
   - Check for sendmail_path configuration

2. **Invalid email address**
   - Verify email format in enrollment form
   - Check `emails/confirmation_log.txt` for validation errors

3. **Email filtered as spam**
   - Check all inboxes and spam folders
   - Check server email queue status

4. **Backup files not saving**
   - Verify `emails/` directory exists and is writable
   - Check file permissions

**Quick Fix:**
- Always check `emails/confirmation_log.txt` for error details
- This file contains complete history of all sending attempts

## 🚀 Important Configuration

**Gmail Users (Optional SMTP Setup):**
If PHP mail() doesn't work, you may need SMTP configuration:

Update `send_confirmation_email.php` with:
```php
$mail->Host = 'smtp.gmail.com';
$mail->Port = 587;
$mail->Username = 'your-email@gmail.com';
$mail->Password = 'your-app-password';
```

**For local testing, PHP mail() should work if:**
1. Server has sendmail installed
2. `from:` address is valid domain
3. No mail server blocking
